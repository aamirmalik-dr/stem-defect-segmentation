# Data

Everything this project trains and scores on is **synthetic**, generated by
`stemseg.sim`. There is no downloaded dataset and nothing here is derived from
any external corpus. The simulator emits an image together with its exact
per-pixel label map, so every metric in the repository is measured against
ground truth that is known by construction, not annotated by hand.

## Committed sample (`data/sample/`)

Four small `.npz` samples, one per material preset, are committed so the demo
and gallery run with no setup:

| File | Material | Dose (counts/peak) |
|------|----------|--------------------|
| `graphene_d150.npz` | graphene honeycomb | 150 |
| `hbn_d150.npz` | hexagonal boron nitride | 150 |
| `mos2_d200.npz` | MoS2 monolayer | 200 |
| `oxide_d250.npz` | square perovskite-like oxide | 250 |

Each file stores the float32 image, the int8 label map, the disordered-region
mask, the drawn column centres and classes, the vacancy sites, and the
`SegConfig` that produced it. Regenerate them at any time with:

```
stemseg samples
```

The full training distribution is not committed (it is generated on the fly);
`stemseg train` reproduces it from fixed seeds.

## Real images (`data/real/`)

The repository does not commit a real micrograph. The models are trained
purely on simulation, so any output on real data is qualitative (there is no
ground truth to score against). To run on your own image, see
[`data/real/README.md`](real/README.md).
